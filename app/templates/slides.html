My solution


Math.stackexpert

  • Clustered tags into manageable 'categories' representing broad areas of mathematics
  • Used voting data and question text to build a predictor (logistic regression model) for whether or not a question will be closed for lack of effort.

Methods/Analysis

Predictor:

  • Used 4916 questions, half closed.
  • Tuned parameters with cross-validation (optimized for recall)
  • Held back 20% as test set:
    • Accuracy: 85.66%
    • Recall: 87.47%
    • Precision: 84.57%

Clustering:

  • Used sklearn AffinityPropogation algorithm
  • Fed in similarity matrix built from co-occurrence data
